# 💡 RU Keyboard Tips 

## 🎯 Purpose

Practical tips to improve efficiency when using the **Russian keyboard layout on macOS**.

---

## ⚡ Use Shortcuts by Position, Not Letters

Shortcuts in macOS are based on **physical key positions**, not visible characters.

Example:

```text
⌘ + C (Copy) → appears as ⌘ + С in RU layout
```

**Why:**
The shortcut does not change — only the visible letter does.

---

## 🔄 Be Aware of Visual Differences

Many Cyrillic letters look similar to Latin ones, but they are different.

Examples:

* A ≠ А
* C ≠ С
* P ≠ Р

**Why:**
Mixing alphabets can cause bugs in code, search, and authentication.

---

## ⚡ Use Spotlight Regardless of Layout

```text
⌘ + Space → search → Enter
```

**Why:**
Spotlight works independently of the active keyboard layout.

---

## 📂 Open Terminal in Current Folder

### Method 1: Drag & Drop

```bash
cd [drag folder here]
```

### Method 2: Right Click (Automator Service)

Create a workflow to open Terminal in the selected folder.

**Why:**
Avoids manual navigation regardless of layout.

---

## ✍️ Faster Navigation in Terminal

| Action | Shortcut |
|-------|---------|
| Copy | ⌘ + C (⌘ + С) |
| Paste | ⌘ + V (⌘ + М) |
| Select All | ⌘ + A (⌘ + Ф) |

---

## 💻 Switch Layout for Symbol-Heavy Tasks

The Russian layout is optimized for text, not symbols.

**Recommended:**

* RU → writing
* US → coding / terminal / configuration

**Why:**
Symbols like `{ } [ ] < > \` are less accessible in RU layout.

---

## ⚡ Reduce Friction When Switching Layouts

Use:

```text
⌃ + Space
```

**Why:**
Fast switching keeps workflows smooth.

---

## 🧠 Think in Workflows

Instead of isolated actions:

* Combine shortcuts
* Switch layouts intentionally
* Automate repetitive tasks

👉 Example:
Write (RU) → Switch → Code (US) → Run → Switch back

---

## ⚠️ Common Mistakes

* Looking at letters instead of key positions
* Mixing Cyrillic and Latin characters
* Staying in RU layout while coding
* Forgetting Option-based symbols
