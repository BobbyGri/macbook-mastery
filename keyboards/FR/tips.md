# 💡 FR Keyboard Tips 

## 🎯 Purpose

Practical tips to improve efficiency when using the **French (AZERTY) keyboard layout on macOS**.

---

## ⚡ Use Shortcuts by Position, Not Letters

Shortcuts are based on **physical key position**, not visible characters.

Example:

```text id="p3c5mq"
⌘ + A (Select All) → appears as ⌘ + Q on FR
```

**Why:**
The shortcut remains the same, even if the displayed letter changes.

---

## 🔤 Master Dead Keys (Critical)

The FR layout relies heavily on **dead keys** for accents.

Examples:

```text id="3n5c3t"
^ → e → ê
¨ → e → ë
```

**Why:**

* Essential for correct French writing
* Faster than copy-pasting accented characters
* Required in most writing contexts

---

## 💻 Expect Symbol Friction

Many symbols are harder to access than in US layout:

* `{ } [ ] < > \`
* require ⌥ or ⇧ + ⌥ combinations

**Why:**

AZERTY is optimized for writing, not programming.

---

## ⚡ Switch Layout for Symbol-Heavy Tasks

Recommended workflow:

* FR → writing
* US → coding / terminal / configuration

**Why:**

Switching is faster than fighting the layout.

---

## 🔄 Switch Layouts Efficiently

Use:

```text id="8s1y8p"
⌃ + Space
```

**Why:**
Fast layout switching reduces friction in mixed workflows.

---

## 📂 Open Terminal in Current Folder

### Method 1: Drag & Drop

```bash id="r9yogm"
cd [drag folder here]
```

### Method 2: Right Click (Automator Service)

Create a workflow to open Terminal in the selected folder.

**Why:**
Avoids manual navigation and works independently of layout.

---

## ✍️ Faster Navigation in Terminal

| Action        | Shortcut   |
| ------------- | ---------- |
| Move by word  | ⌥ + ← / →  |
| Start of line | Ctrl + A   |
| End of line   | Ctrl + E   |
| Delete word   | ⌥ + Delete |

---

## 🔤 Avoid Mixing Layout Logic

Do not assume:

* QWERT positions
* US symbol locations

**Why:**
FR has a different structure → mental mapping must adapt.

---

## 🔤 Use Character Reference When Needed

If you cannot type a symbol quickly:

→ Use [Character Reference](characters.md)

**Why:**

* Faster than searching online
* Keeps workflow uninterrupted
* Reliable fallback

---

## ⚠️ Common Mistakes

* Looking at letters instead of key position
* Forgetting dead key sequences
* Trying to code in FR without switching
* Assuming US layout behavior
